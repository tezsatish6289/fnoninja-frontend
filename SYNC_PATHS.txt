# fnoninja-frontend → tez-terminal sync-back manifest.
#
# POLICY (locked): fnoninja-frontend owns ONLY the fnoninja.com LANDING PAGE
# (marketing). Designers/Lovable edit the landing here; these paths — and ONLY
# these — sync back into tez-terminal (the deploy monorepo) for production.
#
# Everything else is canonical in tez-terminal `main` and is NEVER synced back:
#   - Product UI: /levels, /levels/chart, /subscribe, chat, /learn, /webinar,
#     /contact, /privacy, /terms, /login, /community
#   - Shared chrome: nav, footer, page shells, profile menu
#   - Hooks, shared libs (src/lib/**), firebase, UI primitives
#   - All backend (/api/**, middleware, entitlements-server, zoho)
#
# The export script (tez-terminal → fnoninja-frontend) still copies the full
# tree so the frontend repo stays buildable for Lovable previews; this manifest
# only governs what flows BACK, so landing edits can never clobber product code.
